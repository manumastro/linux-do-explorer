# 做了个小工具：一键把 Linux.do 帖子导出成带图 PDF，直接喂给 AI

最近一直有个很实际的需求：

Linux.do 上有很多长帖，信息密度很高，评论区里也经常藏着补充说明、示例、图片和关键回复。想把这些内容丢给 Claude / ChatGPT / Gemini 继续整理、总结、提问，结果经常卡在第一步：**不好喂**。

最常见的问题有几个：

- 直接复制粘贴，格式容易乱，图片也没了
- 很多 AI 侧边栏工具其实只能读文字，图里的东西它看不到
- 帖子一长，楼层一多，手动整理非常折磨
- 明明只想保存一个好帖，最后却变成一次低效体力活

所以我做了个小工具：

**在 Linux.do 帖子页点一下扩展，就能把当前帖子导出成 Markdown + 带图 PDF，直接拿去喂 AI。**

## 这东西能干什么

现在这版主要做的是“单帖归档”，不是批量抓站。

它可以：

- 一键导出当前帖子
- 保存 Markdown、原始 JSON、图片资源
- 额外生成一份带图 PDF
- 支持只导出指定楼层范围
- 在扩展面板里显示导出进度
- 导出完成后直接打开输出目录

简单说，就是把“帖子 → 可保存、可转发、可喂 AI 的本地资料”这一步变顺。

---

## 我为什么做这个

我自己的使用场景很直接：

看到一个值得留档的帖子，或者一个后续还想继续问 AI 的长帖，我不想再手动复制、贴图、整理上下文。

我想要的是：

**打开帖子 → 点导出 → 拿到 PDF → 直接丢进 AI 对话框。**

这样一来：

- 想总结帖子内容，可以直接问
- 想让 AI 分析图文内容，也能一起带上
- 想长期留档，也比收藏链接稳很多

## 现在怎么用

### 对普通 Windows 用户

我已经做了一个便携版，尽量做到“不折腾环境”：

1. 下载便携包并解压
2. 双击 `01-start-bridge.cmd`
3. 在 Chrome 的扩展页加载 `browser-extension` 目录
4. 打开 Linux.do 帖子，点导出

### 对愿意自己折腾源码的人

也可以直接跑源码版，适合开发者或非 Windows 环境用户。

## 安全和边界

这个我觉得得提前说清楚。

这工具不是拿来批量抓站的，也不是自动化养号工具。

目前的原则是：

- 只处理你当前打开的帖子
- 不做批量抓取
- 不存账号密码
- 不导出 Cookie
- 本地桥只监听 `127.0.0.1`
- 数据尽量在本地完成处理

它更像是把“你本来就能在浏览器里看到的帖子内容”，整理成一份更好用的本地资料。

所以它的定位一直都比较克制：

**偏个人归档，偏知识整理，不偏爬虫。**
